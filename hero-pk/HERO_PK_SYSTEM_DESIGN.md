# 英雄 PK 系统设计稿（V0.1）

## 1. 设计目标
1. 1v1 回合制，双方轮流行动。
2. 每回合最多执行一种攻击行为：主动武功 或 普通攻击，二者互斥。
3. 角色成长围绕一级属性（体力、武力、防御、内力）与二级战斗属性展开。
4. 武功分主动、被动：
   - 主动：伤害型 / 增益型（二选一执行）
   - 被动：开场触发，固定提升能力，并带持续回合数。

---

## 2. 属性体系

### 2.1 一级属性（基础）
- 体力
- 武力
- 防御
- 内力

### 2.2 二级属性（战斗）
- 暴击率
- 暴击伤害率
- 命中率
- 闪避率
- 格挡率
- 格挡伤害（格挡成功时的减伤比例）

### 2.3 战斗用衍生值（建议）
建议在进入战斗时由一级属性计算：

最大生命 = 800 + 40 * 体力

攻击力 = 30 + 6 * 武力 + 2 * 内力

防御力 = 20 + 6 * 防御 + 1.5 * 内力

---

## 3. 回合流程

### 3.1 开场阶段
1. 初始化角色基础属性。
2. 应用所有被动武功效果（进入 Buff/Passive 状态池，带回合数）。
3. 决定先手（可随机或按固定规则）。

### 3.2 行动阶段（每个角色轮到自己时）
1. 结算回合开始触发（可扩展）。
2. 判定是否发动主动武功（满足冷却、内力消耗、触发条件）。
3. 若发动主动武功：执行技能效果，本回合不再普通攻击。
4. 若未发动主动武功：执行普通攻击。
5. 回合结束，减少该角色身上效果剩余回合数；回合数为 0 则移除。

---

## 4. 命中/闪避/暴击/格挡判定顺序

建议顺序：命中判定 -> 暴击判定 -> 格挡判定 -> 最终伤害

### 4.1 命中判定
P_hit = Clamp(基础命中 + 攻方命中率 - 守方闪避率, 5%, 95%)

- 基础命中建议 85%。

### 4.2 暴击判定
P_crit = Clamp(攻方暴击率 - 守方抗暴(可先设0), 0%, 75%)

### 4.3 格挡判定
P_block = Clamp(守方格挡率, 0%, 70%)

---

## 5. 伤害公式设计

### 5.1 普通攻击伤害
1. 原始伤害：
D_raw = 攻击力 * (1 + 随机浮动) - 防御力 * 0.65

- 随机浮动建议区间：[-5%, +5%]

2. 保底伤害：
D_base = max(D_raw, 攻击力 * 0.1)

3. 暴击修正（若暴击）：
D_crit = D_base * (1.5 + 暴击伤害率)

- 其中 1.5 是基础暴伤倍率。

4. 格挡修正（若格挡）：
D_final = D * (1 - 格挡伤害)

- 例：格挡伤害 = 35% 表示格挡后减伤 35%。

### 5.2 主动武功（伤害型）
技能配置参数：倍率系数 A、内力加成系数 B、破防系数 P

D_skill_raw = (攻击力 * A + 内力 * B) - 防御力 * (1 - P)

建议范围：
- A in [1.1, 2.5]
- B in [0.5, 2.0]
- P in [0, 0.5]

后续同样走暴击与格挡结算。

## 6. 武功系统

### 6.1 主动武功
两种模式：

1. 伤害型主动
- 在自己回合释放，释放后本回合不普通攻击。
- 可有冷却（CD）与内力消耗。

2. 增益型主动
- 在自己回合释放，释放后本回合不普通攻击。
- 一次最多增加 5 项属性。
- 每项按固定百分比提升。
- 持续 2 回合（建议按受益者行动结束时扣减）。

示例：
- 攻击力 +20%
- 命中率 +15%
- 暴击率 +10%
- 格挡率 +8%
- 闪避率 +12%

### 6.2 被动武功
- 开场立即生效。
- 效果：固定增加能力值（可百分比或固定值）。
- 必须有持续回合数（建议允许填 -1 表示整场战斗）。

---

## 7. Buff/Passive 叠加规则（建议）

1. 同名效果不叠加，重复施加仅刷新持续回合。
2. 不同名效果可叠加。
3. 属性计算顺序：
最终属性 = (基础值 + 固定加成总和) * (1 + 百分比加成总和)
4. 概率类属性统一上限，避免数值溢出：
- 暴击率、命中率、闪避率、格挡率上限建议 75%。
- 格挡伤害上限建议 60%。

## 8. 数据结构建议（配置化）

V0.3 实现已落地：
1. 角色基础数据已迁移到 `src/main/resources/config/heroes.json`。
2. 武功模板数据已迁移到 `src/main/resources/config/skill-templates.json`（含主动/被动模板、怒气消耗梯度）。
3. Java 代码仅保留战斗逻辑与配置装配，不再硬编码角色或武功参数。
4. 调整角色数量、基础属性、风格、濒死开关、武功倍率、CD、消耗时，优先改配置文件，不改 Java 常量。

当前角色配置结构（数组）：

## 9. 平衡性建议
1. 主动伤害技能平均期望伤害应约为普通攻击的 1.3-1.8 倍。
2. 增益型主动在 2 回合内带来的总收益应接近 1 次高伤主动。
3. 被动持续回合越长，单回合增益应越低。
4. 格挡与闪避不建议同时过高，避免战斗过慢。

---

## 10. 技能分类与体力分段发动率

### 10.1 技能分类
主动武功按分类管理：
- 伤害类（Damage）
- 防御类（Defense）
- 补血类（Heal）

同一回合只允许触发 1 个主动武功；若未触发则执行普通攻击。

修订规则（按角色所学武功）：
1. 不再强制每个角色必须同时拥有伤害/防御/补血三类。
2. 一个角色可以拥有多个同类武功（例如 3 个伤害类 + 1 个防御类）。
3. 技能选择仅在“该角色已学习的武功池”内进行抽取。

### 10.2 体力分段规则（以最大体力 1000 为例）
- 体力 > 500（高血量段）：伤害类优先
- 300 < 体力 <= 500（中血量段）：防御类优先
- 体力 <= 300（低血量段）：补血类优先

### 10.3 基础触发概率模板

全局规则（你最新确认）：
- 每回合主动武功总触发率固定为 15%。
- 普通攻击概率为 85%（以普通攻击为主）。
- 只有在“触发了主动武功”的前提下，才进入下方体力分段类别抽取。

| 体力区间 | 伤害类 | 防御类 | 补血类 |
|---|---:|---:|---:|
| > 500 | 60% | 25% | 15% |
| 301-500 | 30% | 50% | 20% |
| <= 300 | 15% | 25% | 60% |

说明：
1. 每回合先做一次“是否触发主动武功”判定，触发率为 15%。
2. 未触发则直接普通攻击。
3. 触发后，再根据体力区间抽取“技能类别”。
4. 在该类别下按各武功权重抽取具体武功。
5. 若该类别在武功池中不存在或均不可用（CD、内力不足），按概率从其余类别重抽一次。
6. 若重抽后仍无可用技能，再降级为普通攻击。

### 10.4 角色个性修正（可选）
可对模板做小幅修正，体现角色风格：
- 偏进攻角色：伤害类 +5%，其他类别等比缩减。
- 偏稳健角色：防御类 +5%。
- 偏续航角色：补血类 +5%。

## 12. 回合内技能选择伪代码（按体力区间）

```text
输入: hero, hp, maxHp

roll = random(0,1)
if roll > 0.15:
  # 85% 情况：普通攻击
    normalAttack()
  return

# 以下是触发技能后的类别抽取
if hp > 500:
  p = {damage:0.60, defense:0.25, heal:0.15}
elif hp > 300:
  p = {damage:0.30, defense:0.50, heal:0.20}
else:
  p = {damage:0.15, defense:0.25, heal:0.60}

# 可选：叠加角色个性修正后做归一化
p = normalize(applyHeroBias(hero, p))

category = weightedRandom(p)
skill = weightedRandom(availableSkills(hero, category))

if skill exists and isUsable(skill):
  cast(skill)
else:
  # 当前类别不可用可重抽一次；若仍失败，普通攻击
  skill2 = weightedRandom(availableSkills(hero, otherCategories(category)))
  if skill2 exists and isUsable(skill2):
    cast(skill2)
  else:
    normalAttack()
```
