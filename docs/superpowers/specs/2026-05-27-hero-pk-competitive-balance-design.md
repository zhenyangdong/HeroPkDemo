# Hero PK 竞技平衡分支设计（稳定优先，保留少量运气）

## 1. 背景与目标

当前 hero-pk 已具备核心回合制机制（普通攻击、武功释放、怒气、连击/重击、濒死、天气影响等），但在面向“竞技平衡型”演进时，需要明确新的机制目标：

1. 对局主体稳定：大多数回合结果可预期，降低连续随机爆发导致的非策略性波动。
2. 保留少量高光：允许少量秒杀场景，但必须有明确条件链。
3. 弱者可翻盘：在明显差距对局中保留弱者胜强者可能性，但不做保送。
4. 可复现可解释：同 seed 下对局可复盘，关键机制触发有日志证据。

## 2. 方案选择

评估了三类分支方向：

1. 稳态主导 + 条件爆发（推荐）
2. 动态概率修正平衡器
3. 双轨伤害（稳定通道 + 高风险通道）

推荐选择第 1 类，理由：

1. 与目标最一致：稳定优先，同时保留受控爆发。
2. 可解释性更强：高爆发只在窗口期出现，便于日志复盘。
3. 工程风险较低：可在现有 BattleService 流程上增量分层。

## 3. 机制设计

### 3.1 对局分层

对局逻辑分为三层：

1. 常态层（默认）
- 用于收敛伤害和概率波动。
- 保证“竞技稳定”的基本盘。

2. 爆发窗口层（少量回合）
- 在明确触发条件下放宽上限。
- 允许高光回合与低频秒杀。

3. 翻盘保护层（机会型修正）
- 仅在明显劣势场景生效。
- 提供机会，不直接给伤害保底。

### 3.2 数值边界

#### 常态层边界

1. 单次伤害上限：
- 普攻 <= 目标最大生命 16%
- 技能 <= 目标最大生命 24%

2. 概率上限：
- 暴击有效概率 <= 35%
- 连击触发 <= 22%
- 重击触发 <= 16%

#### 爆发窗口边界

1. 触发条件（任一满足）：
- 任一方生命 <= 30%
- 行动方怒气 >= 200

2. 持续：2 回合（按行动方回合计数）

3. 窗口内上限放宽：
- 普攻上限提高到 24%
- 技能上限提高到 40%
- 暴击上限 48%
- 连击上限 30%
- 重击上限 22%

4. 秒杀条件链（全满足才允许）：
- 在窗口期内
- 行动方怒气 >= 260
- 使用高爆发技能且命中
- 秒杀总概率硬上限 3%

#### 弱胜强空间（翻盘保护层）

1. 明显劣势阈值：综合战力差 >= 18%
2. 劣势方机会修正：
- 命中 +3%
- 怒气回复效率 +8%
- 不直接追加固定伤害

3. 胜率目标区间：
- 明显劣势对局中，弱方目标胜率 12%-25%（随差距递减）
- 差距 > 35% 时，强者仍保持明显高胜率（例如 >75%）

## 4. 工程结构设计

### 4.1 Battle 主流程改造

在 BattleService 中引入清晰分层，不改变外部 API：

1. 常态约束层：统一伤害/概率上限
2. 窗口判定层：管理窗口开启、关闭、剩余回合
3. 劣势修正层：按战力差触发机会修正

目标：降低逻辑耦合，后续规则扩展不继续堆叠在单一大方法里。

### 4.2 配置化

新增配置文件：

- src/main/resources/config/balance-rules.json

配置项包括：

1. 常态上限参数
2. 窗口触发条件与持续回合
3. 秒杀条件链参数与概率上限
4. 劣势阈值与机会修正参数

目标：调平衡只改配置，不改核心代码。

### 4.3 结果与日志增强

扩展 BattleResult 元信息与日志字段：

1. 当前回合是否处于爆发窗口
2. 窗口触发原因
3. 劣势修正是否生效
4. 生效参数值

前端增加“平衡日志”折叠区域，默认不干扰主对战视图。

## 5. 验证与验收

### 5.1 验收指标

1. 稳定性：常态局极端短局占比受控。
2. 爆发性：秒杀仅在条件链下出现，且总占比 <= 3%。
3. 逆转性：弱胜强概率落入目标区间，不出现保送感。
4. 可解释性：关键机制触发均有日志证据。
5. 可复现性：同 seed 结果与日志一致。

### 5.2 自动验证清单

1. 构建通过。
2. 固定 seed 黄金样例回归。
3. 批量模拟输出：
- 胜率矩阵
- 回合分布
- 秒杀发生率
- 弱胜强分布

4. 阈值门禁：
- 秒杀率超阈值 -> 阻断
- 极端拖回合超阈值 -> 阻断
- 弱胜强概率越界 -> 阻断

## 6. 分支化实施路线

建议分 4 个功能分支推进：

1. stability-core
- 实现常态约束层
- 引入 balance-rules.json
- 完成核心日志埋点

2. balance-simulator
- 增加批量模拟与报表输出
- 建立阈值门禁

3. burst-window
- 接入爆发窗口
- 上线秒杀条件链

4. underdog-comeback
- 接入劣势机会修正
- 调整到目标胜率区间

推荐顺序：stability-core -> balance-simulator -> burst-window -> underdog-comeback。

## 7. 里程碑计划

1. M1（第 1 周）：稳定内核 + 配置化 + 日志字段
2. M2（第 2 周）：模拟器 + 报表 + 门禁
3. M3（第 3 周）：爆发窗口 + 秒杀条件链
4. M4（第 4 周）：劣势机会修正 + 全量调参复盘

## 8. 风险与对策

1. 规则变多导致可读性下降
- 对策：机制触发标准化日志 + 前端折叠解释面板。

2. 调参震荡
- 对策：每次仅改 1-2 组参数 + 保留基线配置 + 报表对比。

3. 玩家感知“暗箱干预”
- 对策：只做机会型小修正，不做直接伤害保底；在日志中公开触发条件。

## 9. 非目标（本期不做）

1. 新英雄大规模扩容
2. 新资源系统（装备、符文等）
3. 联网实时 PvP 与匹配系统

本期聚焦“竞技平衡分层机制”落地，不扩散到新子系统。
